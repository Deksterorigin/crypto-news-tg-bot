# 📰 Crypto News Telegram Publisher Bot

Полностью автономный Telegram-бот с высокой степенью параллелизма (concurrency), который собирает, фильтрует и публикует новости криптовалют, информацию об активностях (дропах, тестнетах) и ежедневную рыночную аналитику. 

Бот полностью переведен на **асинхронную архитектуру (`AsyncTeleBot`)** и использует интеллектуальный оркестратор **Gemini Free Tier Orchestrator**. Это позволяет ему стабильно работать на **бесплатных лимитах (Free Tier)** API Google AI Studio без блокировок базы данных и ошибок `429 TooManyRequests`.

---

## 🚀 Основные возможности

*   **Асинхронная архитектура**: Полностью переписан на базе фреймворка `AsyncTeleBot` (`pyTelegramBotAPI`). Все сетевые запросы, операции парсинга, работа с СУБД и вызовы API генеративных моделей выполняются в неблокирующем асинхронном режиме.
*   **Оркестратор лимитов Gemini (Free Tier)**: Специальный промежуточный слой (`gemini_orchestrator.py`), который отслеживает лимиты RPM (запросов в минуту) и RPD (запросов в день) и динамически распределяет нагрузку по доступным моделям:
    1.  `gemini-2.5-flash-lite` (Основная: 10 RPM, 20 RPD, интервал между запросами не менее 6 сек)
    2.  `gemini-2.5-flash` (Резервная 1: 5 RPM, 20 RPD, интервал не менее 12 сек)
    3.  `gemini-3.5-flash` (Резервная 2: 5 RPM, 20 RPD, интервал не менее 12 сек)
*   **Отказоустойчивость при ошибках 429 (ResourceExhausted)**: Автоматически перехватывает ошибки превышения квот, применяет экспоненциальную задержку (backoff) с добавлением случайного джиттера и переключается на следующую доступную модель в очереди приоритетов.
*   **Строгая валидация структуры (JSON Schema)**: Использует явные OpenAPI-схемы на базе стандартных словарей (`NEWS_RESPONSE_SCHEMA` и `URGENCY_RESPONSE_SCHEMA`), что предотвращает вырезание обязательных полей (например, `post_text` или `poll`) парсером Google GenAI SDK.
*   **Многопоточная и бесконфликтная база данных SQLite**: База данных переведена в режим WAL (Write-Ahead Logging). Записи синхронизируются через `asyncio.Lock()`, а блокирующие операции чтения/записи вынесены в отдельные потоки через `asyncio.to_thread`. Это полностью решает проблему "database is locked".
*   **Сброс квот с учетом часовых поясов и DST**: Лимиты RPD обнуляются в соответствии с тихоокеанским временем Google (`America/Los_Angeles`) с корректной поддержкой перехода на летнее/зимнее время (PST/PDT).
*   **Семантическое дублирование**: Защита от повторных публикаций одной и той же новости из разных источников на основе коэффициента сходства Жаккара (Jaccard Similarity) по стеммированным токенам и SequenceMatcher.
*   **Многоисточниковый парсер**: Сбор новостей по RSS-лентам, а также кастомный скрапинг анонсов бирж Binance, Bybit, тестнетов с airdrops.io и листингов на CryptoRank.
*   **Ежедневный дайджест**: Автоматический сбор текущих цен валют (через API CoinGecko) и заголовков новостей за день для генерации аналитической колонки.

## 🛠️ Стек технологий

*   **Язык**: Python 3.9+ (Полная совместимость с Python 3.10 – 3.14)
*   **AI SDK**: google-generativeai (Google GenAI Python SDK)
*   **Telegram API**: pyTelegramBotAPI (модуль `AsyncTeleBot`)
*   **База данных**: SQLite (WAL режим + асинхронный запуск в потоках)
*   **Зависимости**: BeautifulSoup4, feedparser, requests, python-dotenv, zoneinfo

## ⚙️ Настройка конфигурации (.env)

Создайте файл `.env` в корневой директории на основе шаблона `.env.example`:

```env
# Токен бота и ID целевого канала Telegram
TELEGRAM_BOT_TOKEN=1234567890:ABCdefGhIJKlmNoPQRsTUVwxyZ
TELEGRAM_CHANNEL_ID=-1001234567890

# API-ключ Google Gemini
GEMINI_API_KEY=AIzaSyYourGeminiApiKeyHere

# Язык публикаций (uk - украинский, ru - русский, en - английский)
POST_LANGUAGE=uk

# Telegram ID владельца (для защиты от несанкционированного управления)
OWNER_ID=987654321
```

## 🕹️ Интерфейс администрирования в Telegram

Бот поддерживает интерактивное меню управления с инлайн-кнопками, доступное владельцу (`OWNER_ID`) и зарегистрированным администраторам:

| Команда | Доступ | Описание |
|---------|--------|----------|
| `/start` | Для всех / Админ | Запуск бота, приветствие и вывод главного меню администратора. |
| `/status` | Админ | Просмотр состояния бота, таймеров, очереди постов и статистики БД. |
| `/list_feeds` | Admin | Вывод списка всех активных RSS-лент и парсеров сайтов. |
| `/add_feed <название> <url>` | Admin | Динамическое добавление нового источника. |
| `/delete_feed <url>` | Admin | Удаление источника из базы данных. |
| `/post_now <news/activity>` | Admin | Принудительный запуск сборщика и генерации поста через AI немедленно. |
| `/daily_analysis` | Admin | Принудительная генерация дневного аналитического дайджеста. |
| `/add_admin <user_id> [username]` | Владелец | Добавление нового администратора бота. |
| `/delete_admin <user_id>` | Владелец | Отзыв прав администратора. |
| `/list_admins` | Владелец | Просмотр списка всех администраторов. |
| `/backup_db` | Владелец | Отправка резервной копии базы данных (`data.db`) владельцу в чат. |

## 📦 Установка и запуск

1.  **Клонируйте репозиторий**:
    ```bash
    git clone https://github.com/Deksterorigin/crypto-news-tg-bot.git
    cd crypto-news-tg-bot
    ```

2.  **Установите зависимости**:
    ```bash
    pip install -r requirements.txt
    ```

3.  **Настройте переменные окружения**:
    ```bash
    cp .env.example .env
    # Откройте .env и заполните конфиг своими ключами
    ```

4.  **Запустите бота**:
    ```bash
    python bot.py
    ```

5.  **Запуск проверочных тестов**:
    *   Проверка отправки сообщений в канал: `python test_post.py`
    *   Проверка работы парсеров: `python test_fetcher.py`
    *   Проверка генерации контента через ИИ: `python test_processor.py`
    *   Запуск полного пакета юнит-тестов: `python -m unittest test_features.py`
